# Git Semantic Memory

This file stores detailed memory about recent Git-based development context.

## Scope

Read this file only when understanding recent project changes.

## Criticality

7

## Notes

- Git memory should be summarized, not dumped.
- `sync --recent <n>` updates the Recent Development Memory section.
- Raw commit logs should not be inserted into `PROJECT_MEMORY.md` by default.
- Git memory uses local `git log` only and does not require API keys.
- Non-Git directories should skip Git memory without failing.
