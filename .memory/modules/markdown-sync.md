# Markdown Safe Sync

This file stores detailed memory about marker-based sync behavior.

## Scope

Read this file only when editing sync behavior.

## Criticality

10

## Notes

- Auto-generated sections must use markers.
- `PROJECT-SCAN` and `GIT-MEMORY` are independent marker sections.
- Manual notes outside markers must be preserved.
- Normal `sync` should not overwrite human-written content.
- Normal `sync` does not rewrite adapter files unless `--force` is passed.
- This is a high-trust feature. Breaking it damages the product.
